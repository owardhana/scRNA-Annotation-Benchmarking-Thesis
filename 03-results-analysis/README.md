# Results Analysis

Statistics and figure-generation scripts for each reported phase. No result data or figures are
included — only the `Plots_Analysis.R` scripts that consume phase-level CSV/TSV output tables
(generated by running `02-benchmarking-pipeline/`) and produce the manuscript's figures/tables.

See the [top-level README](../README.md#manuscript-phase-mapping) for the full folder → phase
mapping table. Summary:

| Folder | Was (original phase code) | Supports |
|---|---|---|
| `synthetic-L9-design/` | P4 Synthetic Datasets V1 | Phase 1 |
| `real-data-validation/` | P5 Real Life Datasets V2 | Phase 2 |
| `cross-platform-transfer/` | P9 Cross Dataset | Phase 3 |
| `marker-synthetic-supplement/` | P6a Marker - Synthetic | Phase 1 (marker-only supplement) |
| `marker-real-supplement/` | P6b Marker - Real | Phase 2 (marker-only supplement) |
| `database-marker-tools/` | P7a Marker - Real Database | Phase 4 |
| `llm-ontology-scoring/` | P7b LLM - Real Internet | Phase 4 |
| `foundation-models-synthetic/` | P8a FM - Synthetic | Phase 5 (supplementary arm) |
| `foundation-models-real/` | P8b FM - Real | Phase 5 (reported arm) |
| `supplementary-figures/` | _SI_generated | Cross-phase supplementary figures |

## Shared code

- **`cluster_dependent_helper.R`** — sourced by every `Plots_Analysis.R` (as
  `source("../cluster_dependent_helper.R")`); annotates tools whose annotation step required a
  ground-truth cluster vector (the dagger-mark convention used in the figures).
- **`shared/score_cells.py`** — LLM-based ontology-aware scoring (0/0.5/1) of predicted vs.
  ground-truth cell-type label pairs, used by both `database-marker-tools/` (Phase 4 database
  arm) and `llm-ontology-scoring/` (Phase 4 LLM arm). Originally two near-identical per-phase
  copies differing only in hardcoded I/O paths; consolidated here into one script parameterized
  via `--raw-dir`/`--out-dir`/`--checkpoint-prefix`. Reads `OPENROUTER_API_KEY` and
  `OPENAI_API_KEY` from the environment.

## Note on data inputs

Every `Plots_Analysis.R` reads phase-level result CSVs (produced by
`02-benchmarking-pipeline/run_benchmark.R`) and, in most phases, a dataset-profile CSV from
`01-data-generation/dataset-profiling/`. None of that data is included in this archive; paths use
`Sys.getenv(...)`-configurable defaults where the original hardcoded a local path, documented
inline in each script.
